## Step 1 — Push to GitHub

```bash
cd gopalcycles-site

git init
git add .
git commit -m "initial: gopalcycles site"

# Create repo on GitHub (go to github.com → New repository → gopalcycles-site)
# Then:
git remote add origin https://github.com/YOUR_USERNAME/gopalcycles-site.git
git branch -M main
git push -u origin main
```

---

## Step 2 — Deploy to Cloudflare Pages

1. Go to **Cloudflare Dashboard → Workers & Pages → Create → Pages**
2. Click **Connect to Git** → select your `gopalcycles-site` repo
3. Set build settings:

| Setting | Value |
|---|---|
| Framework preset | **Astro** |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Node version (env var) | `NODE_VERSION = 22` |

4. Click **Save and Deploy**

Cloudflare will build and deploy. First deploy takes ~2 minutes.
You'll get a URL like `gopalcycles-site.pages.dev` — that's your live preview.

---

## Step 3 — Connect your custom domain (gopalcycles.com)

Since your domain is already on Cloudflare DNS, this is one click:

1. In the Pages project → **Custom domains → Add custom domain**
2. Enter `gopalcycles.com` (and optionally `www.gopalcycles.com`)
3. Cloudflare auto-creates the DNS record — no manual CNAME needed
4. SSL is automatic

For `www` redirect to apex (or vice versa), add a redirect rule in **Cloudflare → Rules → Redirect Rules**:
- Match: `www.gopalcycles.com/*`
- Redirect to: `https://gopalcycles.com/$1` (301)

## Step 4 — Wire Brevo

### 4a. Get your Brevo keys

- Brevo Dashboard → **Settings → API Keys** → copy your API key
- Note your **List ID** for the newsletter subscribers list (Contacts → Lists)

### 4b. Add a Cloudflare Worker to proxy Brevo API calls

**Never put your Brevo API key in frontend code.** Create a Worker:

```js
// brevo-proxy/index.js
export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    const path = url.pathname;

    if (request.method !== 'POST') {
      return new Response('Method not allowed', { status: 405 });
    }

    const body = await request.json();
    const corsHeaders = {
      'Access-Control-Allow-Origin': 'https://gopalcycles.com',
      'Content-Type': 'application/json',
    };

    // POST /subscribe — add contact to Brevo list
    if (path === '/subscribe') {
      const res = await fetch('https://api.brevo.com/v3/contacts', {
        method: 'POST',
        headers: {
          'api-key': env.BREVO_API_KEY,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          email: body.email,
          listIds: [parseInt(env.BREVO_LIST_ID)],
          updateEnabled: true,
        }),
      });
      const data = await res.json();
      return new Response(JSON.stringify(data), { status: res.status, headers: corsHeaders });
    }

    // POST /contact — send transactional email notification to you
    if (path === '/contact') {
      const res = await fetch('https://api.brevo.com/v3/smtp/email', {
        method: 'POST',
        headers: {
          'api-key': env.BREVO_API_KEY,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          sender: { name: 'GopalCycles Contact', email: 'hello@gopalcycles.com' },
          to: [{ email: 'hello@gopalcycles.com' }],
          replyTo: { email: body.email, name: body.name },
          subject: `New contact: ${body.name} — ${body.fleet || 'no fleet specified'}`,
          htmlContent: `
            <p><strong>Name:</strong> ${body.name}</p>
            <p><strong>Email:</strong> ${body.email}</p>
            <p><strong>Fleet:</strong> ${body.fleet || '—'}</p>
            <p><strong>Message:</strong><br>${body.message}</p>
            <p><strong>Subscribe to newsletter:</strong> ${body.subscribe ? 'Yes' : 'No'}</p>
          `,
        }),
      });

      // Also add to list if they opted in
      if (body.subscribe) {
        await fetch('https://api.brevo.com/v3/contacts', {
          method: 'POST',
          headers: { 'api-key': env.BREVO_API_KEY, 'Content-Type': 'application/json' },
          body: JSON.stringify({
            email: body.email,
            attributes: { FIRSTNAME: body.name },
            listIds: [parseInt(env.BREVO_LIST_ID)],
            updateEnabled: true,
          }),
        });
      }

      const data = await res.json();
      return new Response(JSON.stringify(data), { status: res.status, headers: corsHeaders });
    }

    return new Response('Not found', { status: 404 });
  }
};
```

**Deploy this Worker:**
```bash
# In a separate folder (brevo-proxy/)
npm init -y
npx wrangler deploy index.js --name gopalcycles-brevo
```

**Set secrets:**
```bash
npx wrangler secret put BREVO_API_KEY
npx wrangler secret put BREVO_LIST_ID
```

### 4c. Update the contact page

In `src/pages/contact/index.astro`, find `const WORKER_URL = 'https://YOUR_WORKER.workers.dev'`
and replace with your actual Worker URL (e.g. `https://gopalcycles-brevo.YOUR_SUBDOMAIN.workers.dev`).

Then uncomment the `fetch()` calls in the form handlers (the `// Option A` block).

### 4d. Brevo client-side tracking (optional)

In `src/pages/contact/index.astro`, find `client_key: "YOUR_BREVO_CLIENT_KEY"` and replace
with your actual Brevo tracker key (Brevo → Settings → Tracking code).

This enables Brevo's page tracking and contact identification for marketing automation.
